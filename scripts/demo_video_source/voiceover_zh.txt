AI Agent 已经越来越接近真实的链上金融执行。
但在它真正下单之前，我们需要一层安全检查：理解交易意图，绑定证据，拦截高风险行为，并留下可以审计的证明。

InjectiveLens Agent Guard 是一个面向 Injective 上自主交易 Agent 的执行前安全与证明层。
它不保管私钥，不自动签名，不转账，也不直接下单。
它做的是在执行之前，先审查这一步操作是否安全。

这里 Agent 收到一个自然语言交易请求：
用 60% 的可用保证金，开一个 10 倍杠杆的 INJ 永续多单。
Guard 会把这句话解析成结构化交易意图，读取配置好的 Injective 证据来源和明确标注的 demo fixture，然后判断这一步是否应该被允许执行。

结果是 BLOCK。
因为 10 倍杠杆和 60% 保证金占用，对于默认自主执行路径来说风险太高。
这个决定不是随便拦截，而是绑定到 6 条证据，并生成一个 assessment hash。

Guard 不只是拒绝操作。
它会解释为什么：杠杆风险、保证金占用风险、清算距离风险，以及证据覆盖缺口。
然后它会模拟一个更安全的替代方案：把杠杆降到 3 倍，把保证金占用降到 15%，并且仍然要求人工确认之后才能继续执行。

Audit 页面展示的是执行边界。
Guard 允许只读查看证据、模拟更安全的交易、记录 assessment hash，以及验证 proof。
但它会阻止下单、自动签名、转移资金，以及由 LLM 直接生成并执行交易。
LLM 可以解释原因，但真正执行边界由确定性策略来强制。

最后，系统生成 assessment hash。
记录和验证 proof 都需要明确确认，所以应用不会静默签名，也不会悄悄下单。
这让 Agent 的行为在进入链上执行之前，可以被审查、复核和验证。

InjectiveLens 的目标不是替代交易 Agent，而是在 Agent 进入链上执行之前，加上一层可审计、可验证、不会自动签名的安全防护。
